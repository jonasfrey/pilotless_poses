Write a JavaScript function body. The argument o_img holds the image and its pose data. Return false (or any falsy value) to filter the image out of the preview; return true to keep it.

function (o_img, f_deno_write_file, f_save_image, f_save_json, f_save_filtered, lil_gui, f_gui) {
}
What's available on o_img?
  • o_img.s_path_abs — absolute path to the source image
  • o_img.s_name_file — just the file name
  • o_img.s_msg_error — error message if inference failed
  • o_img.n_scl_x, o_img.n_scl_y — image width / height in pixels
  • o_img.n_persons — number of detected people (= o_img.a_o_person.length)
  • o_img.a_s_tag — array of tags you've put on this image in the preview (type a tag in "Active tag", press m / click ★). E.g. ["marked_hands_in_the_air"]. See the has_tag processor.
  • o_img.a_o_person — array of person objects (o_person)

Each o_person has keypoints grouped by body part. Paired parts expose o_left / o_right; the nose sits directly on the person:

  • o_person.o_nose
  • o_person.o_eye, o_ear, o_shoulder, o_elbow, o_wrist, o_hip, o_knee, o_ankle — each with .o_left and .o_right
  • Every point is { n_x, n_y, n_conf }, e.g. o_person.o_wrist.o_left.n_x (or null if the keypoint is absent)

You also receive these file helpers (all return a Promise resolving to whether the write succeeded; paths are relative to the server working directory):

  • f_save_image(dir) — copy this image into dir
  • f_save_json(dir) — write this image's pose JSON into dir as <name>_pose.json
  • f_save_filtered(dir) — do both (image + pose JSON)
  • f_deno_write_file(path, data) — low-level Deno.writeFile proxy; data may be a string, Uint8Array, ArrayBuffer, or number[]

And these UI helpers for adding controls to the Processor controls panel on the preview page (under the Pipeline column):

  • f_gui(label, default) — render a persistent control (text / number / checkbox, by the default's type) and return its current value. The value is remembered across images and runs, so users can type a setting (e.g. a destination folder) once. The control appears the first time a run reaches it.
  • lil_gui — the raw lil-gui namespace for advanced panels, e.g. new lil_gui.GUI() (null if lil-gui can't load offline)
